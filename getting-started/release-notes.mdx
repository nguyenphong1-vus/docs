## Overview

**Release Notes** ghi lại toàn bộ thay đổi của OVI Design System theo từng phiên bản — bao gồm component mới, cải tiến, sửa lỗi, và breaking changes. Đây là tài liệu tham chiếu chính thức để đội ngũ Design – Engineering biết **điều gì thay đổi**, **khi nào**, và **cần migrate ra sao**.

Hệ thống tuân thủ [**Semantic Versioning**](https://semver.org/) với định dạng `MAJOR.MINOR.PATCH`:

- **MAJOR** — breaking change, yêu cầu migration.
- **MINOR** — thêm tính năng/component mới, backwards-compatible.
- **PATCH** — sửa lỗi, không ảnh hưởng API.

## Application

### Cách đọc Release Notes

Mỗi entry tuân theo cấu trúc:

- 🚀 **Added** — tính năng/component mới
- ✨ **Improved** — cải tiến component có sẵn
- 🐛 **Fixed** — sửa lỗi
- ⚠️ **Breaking** — thay đổi yêu cầu migration
- 🗑️ **Deprecated** — sắp loại bỏ, có thay thế

### Cách áp dụng cho Web

1. Kiểm tra phiên bản đang dùng: `npm list @ovi/ui-web`.
2. Đọc release notes giữa version hiện tại và version mới nhất.
3. Cập nhật: `npm update @ovi/ui-web@latest` (cho MINOR/PATCH) hoặc theo migration guide (cho MAJOR).
4. Chạy visual regression test trước khi deploy.

### Cách áp dụng cho App

1. Kiểm tra phiên bản: `npm list @ovi/ui-mobile`.
2. Cập nhật token trước, component sau — tránh xung đột style.
3. Test trên cả iOS & Android, chú ý đặc biệt các thay đổi về animation và safe area.
4. Ship qua **OTA (CodePush)** nếu chỉ là PATCH; cần build mới cho MINOR/MAJOR có thay đổi native.

## Do's & Don'ts

| ✅ Do | ❌ Don't |
| --- | --- |
| Đọc release notes trước khi update version | Update mù vì "thấy có version mới" |
| Cập nhật MINOR/PATCH thường xuyên (≤ 2 tuần) | Để version tụt hậu \> 6 tháng rồi update một lần |
| Theo dõi tag `Deprecated` để chủ động migrate | Đợi component bị xoá mới bắt đầu sửa |
| Báo cáo regression qua issue tracker với version cụ thể | Báo "không hoạt động" mà không nêu version |
| Lock version trong production (`package-lock.json`) | Dùng `^` cho MAJOR trong production |
| Test breaking change trên staging trước | Deploy thẳng lên production sau MAJOR update |
